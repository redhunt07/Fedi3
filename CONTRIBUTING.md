# Contributing to FEDI3

Thank you for contributing to FEDI3.

This project exists to protect decentralization, privacy and protocol unity.

## License

All contributions are released under AGPLv3.

## Forks

Forks are encouraged for:

- Performance improvements
- Security enhancements
- UI/UX customization
- Accessibility
- Experimental features

Forks must preserve protocol compatibility whenever possible.

## Code Rules

- Preserve original copyright headers.
- Add your own author line when modifying files.
- Do not remove existing attribution.
- Keep docs in sync with behavior.
- Prefer small, reviewable PRs.

## Network Responsibility

Any change affecting protocol, federation or P2P behavior must be documented.

Intentional fragmentation is discouraged.

## UI Freedom

UI and graphical customization is fully encouraged.
UI forks must not silently alter protocol behavior.

## Development

Quick checks:

- `cargo check -p fedi3_core`
- `cargo check -p fedi3_relay`
- `cd app && flutter analyze`

## Ethics

FEDI3 forbids contributions that introduce:

- Tracking
- User profiling
- Advertising systems
- Data resale mechanisms
